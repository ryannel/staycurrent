# Gap Ledger

> Working file. Each brownfield phase appends the divergences from GroundWork standard it discovers. `groundwork-infra-adopt` consolidates this into the roadmap table of `docs/maturity.md` at the end of setup (model: `.groundwork/skills/maturity-model.md`), and the bet loop reads it when planning every bet.

Each entry records what diverges, the standard it violates, a severity, and a recommendation.

**Severity:** `blocks-delivery` | `standard-divergence` | `cosmetic`
**Recommendation:** `fix-now` | `defer` | `blocks-delivery`

Definitions: `.groundwork/skills/maturity-model.md` — write them exactly; a drifted spelling orphans the row.
