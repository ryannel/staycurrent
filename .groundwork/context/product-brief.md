# Downstream Context — Product Brief

### Key Decisions

- Two products, one engine: open-source framework + first site (staycurrent.dev)
- Surfaces: reader-facing website (MVP); operator workbench inside Claude Code (MVP)
- A topic = living article + companion AI skill; one stance, two renderings
- Version history, standalone changelog, and per-version provenance are first-class content
- Research loop is scheduled, operator-convened, conversational; significance decision gates version cuts
- Skill updates are pull; RSS is the site's only push channel
- Topics are broad practice areas (e.g. testing, observability, cloud cost engineering)
- Operator is the first user; readership beyond them is upside, not survival

### Binding Constraints

- Research executes in the operator's Claude Code session on subscription pricing; no hosted autonomous service
- Every version publishes with a sources-and-influences provenance record; synthesis labelled as synthesis
- Article and skill never disagree; both cut in the same version
- No approval gate after the version cut; auditability is the compensating control
- One operator per site

### Deferred Questions

- Skill packaging and distribution format — resolved in Architecture
- First topic set and seeding from existing corpus — resolved in MVP Planning

### Out of Scope

- News aggregation
- Community features (comments, accounts, social)
- Multi-author editorial workflow
